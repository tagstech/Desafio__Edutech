    Flexbox

    O Flexible Box Module , geralmente chamado de flexbox , foi projetado tanto como um modelo de layout unidimensional quanto como um método capaz de organizar espacialmente os elementos em uma interface, além de possuir recursos avançados de alinhamento. Este artigo artigos um resumo das principais características do flexbox , como quais exploraremos com mais detalhes não existem deste guia.


    Quando se enunciados o flexbox como sendo unidimensional, enfatiza-se o fato de que ele lida com o layout em uma dimensão de cada vez - seja uma linha ou uma coluna. Isto pode ser comparado com o modelo bidimensional de CSS - Layout de Grade , que permite o controle simultâneo das colunas e linhas.



    Contêiner Flex

    Uma área de um documento que faz uso do flexbox é chamada de contêiner flex . Para criar essa estrutura, defina o valor da propriedade display do elemento representado pelo contêiner como flex ou inline-flex. Desse modo, os elementos-filhos desse contêiner tornar-se-ão do tipo flex . Como todas as propriedades sem CSS possuem valores padrão, ao criar um contêiner flex, os elementos nele contidos apresentar o seguinte comportamento:


  • Exibição em linha (o padrão do flex-direction é row).
  • Alinhamento na borda inicial do eixo principal.
  • Não há expansão no eixo principal, mas pode haver contração.
  • Haverá expansão vertical para preencher a altura do eixo transversal.
  • Uma propriedade flex-basis (en-US) disponível definida como auto.
  • A propriedade flex-wrap é definida como nowrap.

    • O resultado final é que todos os elementos devem ser adequados em uma linha, usando o tamanho do conteúdo como o tamanho no eixo principal. Se houver mais itens do que for possível caber no container, não haverá uma quebra de linha; ao incorrer, exceder o limite horizontal da página. Se alguns elementos principais mais altos que outros, todos os itens se estenderão ao longo do eixo transversal para preencher seu tamanho total.

    Propriedade flex-direction


    A propriedade flex-direction permite alterar a direção dos elementos flex eliminados ao longo do eixo principal. Definindo a propriedade flex-direction como row-reverse(linha reversa) ainda teremos os elementos dispostos em uma linha, entretanto, as linhas inicial e final serão trocadas.


    Se mudarmos flex-direction para uma column(coluna), o eixo principal exibirá o elemento em uma coluna. Trocando o valor para column-reverse(coluna reversa) fará com que as linhas inicial e final sejam novamente trocadas.


    Nenhum exemplo prático abaixo tem-se uma propriedade flex-directiondefinida como row-reverse. Experimente os outros valores - row, column e column-reverse - para ver o que acontece com o conteúdo.
